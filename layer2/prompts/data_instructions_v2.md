════ HARD RULES R1–R14 — READ FIRST. Each is deterministically GATE-ENFORCED: a violating field/leaf is blanked, reverted or rejected and never ships. Later sections defer to these; none is restated. ════

R1. SLOT VERBATIM (the #1 correctness rule) — every data field's `slot` MUST be COPIED VERBATIM from the "FILLABLE DATA-LEAF SLOTS" block (the exact dotted/indexed leaf path, e.g. `health.data.phases[0].value`, `flow.vm.kpis.lossKw`). An invented token (tile_r, kpi_actual, series_r, sourceInputKw, instantaneous_power) resolves to NOTHING — the field, even with a perfect column, is DISCARDED and the leaf stays empty. NEVER emit a slot not on that list. ONE field per leaf; a per-phase/per-series leaf gets one field per element path ([0].value, [1].value, …), each bound to THAT phase's column (R→*_r, Y→*_y, B→*_b, N→*_neutral).

R2. HONEST-BLANK = OMIT THE FIELD (the ONE honest-blank convention; the #1 rule of this seam) — when NO basket column (and no legal fn per R14) measures a slot's OWN quantity — including the R6 walls and off-domain quantities this meter class does not measure (battery SOC / battery temperature / fuel level on an electrical meter) — emit NO field for that leaf AT ALL: the skeleton preserves the key, the leaf renders an honest blank with a per-leaf reason, data_note says why, answerability="partial". You MUST NOT instead: (a) bind a proxy fn/column of a DIFFERENT quantity (loadFactorPct into a "transfer readiness score"; activeEnergyTodayKwh into a "transfers count / days-since" slot); (b) ship a CONSTANT stand-in; (c) REUSE a value from another slot; (d) emit a raw/bucketed field with column=null (the gate defect 'missing a resolved column', not a blank — kind="time" is the ONLY legitimately column-less kind); (e) honest-blank via a mislabelled const. The CONVERSE is the #1 defect: do NOT omit a leaf when the schema HAS a matching column — that blanks a card that HAS data.

R3. NEVER FABRICATE + COUNTABLE COVERAGE — never fabricate a column, number, fn, metadata key or frame field, not even to force answerability "full" (a substitute is a REAL column; a true gap is reported as "none"). Every listed slot is either (a) bound by EXACTLY ONE field or (b) deliberately omitted per R2 WITH the reason in data_note — an unexplained uncovered slot is recorded as a per-leaf gap; never silently ignore a slot, never pad coverage. NEVER A SILENT ZERO-SKELETON: binding NOTHING (fields [] and no roster) with no honest reason renders the skeleton's 0/[] placeholders as real values (a false "0 issues" story) — whenever you bind nothing, declare answerability "none"/"partial" AND a data_note naming exactly WHY; your authored reason beats the generated one.

R4. QUANTITY WALL — every DB SCHEMA line shows the column's `qty=`, every slot line its `expected_qty=`: a column of ONE physical quantity NEVER fills a slot of ANOTHER. power/energy is NOT a temperature (hotspotC/oilC/windingC), an aging factor (faa), a readiness/score, a count (tapCount/transfers), a duration (runHours/backup minutes), or a lifetime (lifeYears); a deviation/spread column is NEVER crest-factor/flicker (flickerPst); thd_current_* is NEVER a voltage-harmonic (h5/h7) or voltage-THD value. SHARING A UNIT DOES NOT MAKE TWO QUANTITIES THE SAME: a voltage-THD leaf (%) never takes thd_current_* (also %); SOC % never takes load %; a `years`/`°C` leaf never takes an energy column/fn (how "10,156 years" happens). Match the slot's LABEL SEMANTIC (voltage vs current, energy vs years), not the unit sign; a slot marked `(weak)` is dimension-only — its label decides. No column of the slot's quantity → R2; the gate blanks every cross-quantity bind ("<quantity> not measured by this meter"), and such a leaf is never "full".

R5. SEMANTIC-FAMILY WALL (the same-DIMENSION pun) — a slot whose label/metric/key names efficiency / SFC / specific consumption / fuel binds ONLY a source of that SAME semantic family — sharing '%' does not make a load factor an efficiency (loadFactorPct into an 'Efficiency' KPI rendered "Efficiency 5.3 %"). No same-family source (a DG with no fuel-flow measurement cannot measure efficiency/SFC) → R2, naming the missing family in data_note.

R6. THE FIVE PHYSICAL WALLS — quantities the meter does NOT log; NO fn fabricates them from unrelated columns; always R2, never a derived field: (1) waveform shape (crest_factor / flicker_pst); (2) per-order harmonics (harmonic_5th/7th_pct, k_factor); (3) externally-assigned nameplate/commercial figures (kpi_kw_load_pct_of_rated, rated_kva, subsidy/contract) when the base_columns are absent OR this asset's nameplate denominator is empty; (4) device run-state (breaker_state, *_trend_status); (5) TRANSFER/READINESS/SYNC/PERMISSIVE/BYPASS SCORES and TRANSFER COUNTS / DAYS-SINCE / LIFETIME — no fn turns a load factor or an energy counter into a readiness score or a transfer count.

R7. THE CANONICAL SAME-QUANTITY-FAMILY PROXY RULE (the ONE exception to locked display metadata — stated once, here): a substitute/proxy is legal ONLY within the slot's physical-quantity family — the bound column's `qty=` EQUALS the slot's `expected_qty=`: a coarser aggregate or adjacent form of the SAME measure ('per-phase kW' absent → total kW ✓; today's kWh absent → this-week kWh ✓; import register dead on a reversed-CT feeder → the EXPORT register delta ✓). SAME family: bind it (kind=raw/derived on a REAL column — NEVER a valueless const) + declare it in `data_note` + MORPH every caption/label/window leaf that DESCRIBES that slot to the proxy's TRUE meaning (e.g. `sourceInputCaption` 'SOURCE INPUT'→'MEMBER IMPORT', 'today'→'this week'), each listed in `_morphed` (siblings sharing the leaf → morph the per-slot caption instead). DIFFERENT family: FORBIDDEN, morphed or not — no data_note legitimizes it → R2. A declared proxy missing its display morph is gate-DROPPED (the leaf blanks rather than show a wrong reading under the default caption); an undeclared proxy is a UNIT-MISMATCH defect. PLAN PROXIES FIRST: exact_metadata is emitted BEFORE data_instructions — decide every proxy before authoring the metadata.

R8. NO BINDING REUSE ACROSS DISTINCT SCALARS — NEVER bind the SAME fn or the SAME column into ≥2 DISTINCT scalar VALUE slots naming DIFFERENT quantities (three "score" cells ups_input/bypass/sync_permissive_score; three "activity" cells days-since / count-30d / lifetime): copying one measurement into all three FABRICATES three distinct readings (they all render 96.3). Bind each scalar ONLY to the source measuring ITS quantity; the rest → R2, never padded. (LEGITIMATE: a SERIES and its own legend/kpi/axis-min/max summary share the series' column — one quantity across related slots; the defect is ≥2 DISTINCT-quantity SCALAR cells co-bound with NO co-bound series.)
    R8-ROLE — SAME QUANTITY, DIFFERENT SOURCE ROLE is the same defect: a slot naming a DEDICATED SENSING POINT (a UPS `bypass` voltage/frequency, an `output`/load or `battery` reading) needs a column measured AT that point. This meter senses only the INPUT/line — voltage_avg / frequency_hz ARE the input reading. Binding voltage_avg into BOTH inputVoltageV AND bypassVoltageV shows the line reading twice, once fabricated as a bypass the meter cannot see (a same-quantity role smear the quantity wall R4 does NOT catch — voltage↔voltage is compatible). Bind input* to the plain reading; every bypass*/output*/battery* slot with no dedicated column → R2 honest-blank (the gate blanks it: "this meter has no bypass column"). A time-axis label leaf (points[*].label / .slot) is ALWAYS kind=time (no column) — a measured column there is R2/gate-blanked (the c59 negative-kW-as-time-labels defect).

R9. SIBLING-SLICE + PHASE-AS-GROUP PROXY — sibling slices carrying DIFFERENT entity labels (breakdown[0..2].value labelled "UPS"/"BPDB"/"HHF"; every sections[i].totalKw; a sankey's nodes[i].value and links[i].value; sources/consumers[i].meters[j]) each claim a DIFFERENT sub-group's number: ONE panel-total column bound into all renders the same value N times and the groups sum to N× the panel (the 'every ribbon = 550,320 kWh' defect). The EVASION is equally forbidden: per-PHASE columns (*_r/*_y/*_b/*_neutral) bound one-each into GROUP-labelled slices — a PHASE is NOT a feeder group / member set / section; a per-phase column fills ONLY a phase-labelled leaf. No per-group column → bind the TOTAL slot only, slices unbound (R2) + answerability="partial" + data_note — on a PANEL asset (see the PANEL MEMBERS block) the panel-aggregate renderer fills each per-member leaf from that member meter's OWN row, so the blank still renders real per-member truth. (DISTINCT: the same column across TIME samples of one series is legitimate.)

R10. CONST-SOURCE — a NUMERIC const in a data slot ships ONLY when the field itself cites a REAL DB source: EITHER (a) a NAMEPLATE RATING — `metric` is exactly one of contracted_capacity_kw / rated_kw / rated_kva / rated_current_a; the executor substitutes this asset's REAL rating or honest-blanks the leaf; the baked `value` is only a shape placeholder (a PER-ASSET rating/contract is NEVER a literal — the default's number is a Storybook seed; a home-made spelling like I_RATED / I_MAX / deratedKva does NOT resolve and is blanked); OR (b) an app_config `consts.<name>` row — `metric` names the row (stress_border_pct, hotspot_warn_c) and `value` IS that row's value. Any other invented numeric literal (a guessed rated current 131, a derated 1000 kVA, made-up thresholds/axis ticks) is gate-blanked; no real source → R2. A UNIVERSAL standard (the IEEE_519 THD limit) is a true literal.

R11. WINDOW/LABEL COHERENCE — the ems_backend range/window you declare, the `window.lookback` you emit, and EVERY period-declaring metadata leaf (`periodLabel`, `range`, a "Monthly"/"Weekly"/"Today" caption — NOT a picker's rangeOptions list) MUST all name the SAME window, the one the consumer actually fills from. A 'Monthly' label over a 24h fill mislabels every number under it (fabrication by caption). Either declare the range the label promises, or MORPH the label leaf to the fetched window's truth (in `_morphed`) — else the gate (policy `gates.window_label_policy`) morphs/blanks it and records the incoherence. Your declared range also DRIVES the backfilled fill window — declare the range the story needs, once, and keep every label consistent with it.

R12. BYTE-IDENTICAL METADATA + `_morphed` — author every exact_metadata leaf BYTE-IDENTICAL to the card's STATIC CONFIG default unless the prompt + 1a story justify a morph; `_morphed` (REQUIRED, inside exact_metadata) lists EVERY changed key path (`[]` when none). The producer applies ONLY declared morphs — an UNDECLARED morph is REVERTED to the default (silent no-op, recorded). HARDCODED/STATIC items are LOCKED, never morphed: legends (statusLegend), colour tokens/palettes/statusColors/selectionColors, axis labels (metricAxisLabels), units, descriptors, vocab, playback glyphs, shade/legend rows (e.g. the heatmap FOOTER's time-axis + shade legend), badge labels — byte-identical ALWAYS, NEVER in `_morphed`. R14. RAW-BEATS-FN; LIBRARY-ONLY FNS; EMPTY DENOMINATOR — check FIRST, before any fn: if a COLUMN BASKET line measures the slot's quantity AND shows data=Y (the ★ REAL LOGGED marker), bind it kind=raw by its EXACT name and emit NO fn — EVEN IF its name_hint says "derived" (name_hint is a spelling guess, not an order: current_max_spread, current_unbalance_pct, current_min/current_max/current_avg are LOGGED columns — READ them raw). A SUMMARY STAT or AXIS-bound slot (a `stats[i].value`, maxY/minY, expectedMax/Min) is ALMOST ALWAYS one of these raw columns, NOT an fn: read the stat's `expected_qty=`/unit and LABEL and bind the same-quantity data=Y raw column it names — "Peak Current"→current_max (kind=raw), "Average Current"→current_avg, a min/minY→current_min, a spread stat→current_max_spread. THE target_column-bridge IS DECISIVE: the instant the frame column you would name as `target_column` is itself a data=Y basket column, STOP — DELETE the fn and re-emit the field as kind=raw with `column`=that exact column (never wrap a real logged column in an fn; {kind:derived,fn:worstPeakKw,target_column:current_max} still blanks on the fn's quantity, {kind:raw,column:current_max} fills). A recovery fn is a LAST resort, legal ONLY when BOTH hold: (a) its `quantity=` tag EQUALS the slot's own quantity (a differing quantity= is a WRONG-QUANTITY bind even if the name sounds close — neutralToPhaseRatioPct is quantity=neutral-to-phase-ratio-percent, WRONG on a current-unbalance slot), AND (b) ALL its base_columns are in the COLUMN BASKET. The ONLY legal `fn` values are the value_keys enumerated VERBATIM in the RECOVERY LIBRARY — a composed fn (currentMaxSpread, maxSpread, currentSpreadPct, …) DOES NOT EXIST and silently blanks the leaf ('derivation unbound'). A fn with a `nameplate:<rating>` base is bindable ONLY when this asset's rating is ACTUALLY populated — an empty rating is NEVER a denominator (→ R2). No fn satisfying (a)+(b) → R2; never the closest-sounding fn.

════ PART 1 — KEEP / SWAP ════
Decide whether the card in this slot tells its assigned analytical story, or whether one of this slot's SWAP CANDIDATES (same footprint, ±15% in width AND height) tells it clearly better. SCOPE: judge RELEVANCE-TO-THE-STORY + COHERENCE only — you do NOT judge whether the data exists or the query will succeed (data_instructions + the helper handle fillability). Be CONSERVATIVE: KEEP is the default; 0 swaps is a correct, healthy answer. A swap is honored ONLY if ALL THREE hold:
1. STORY-ANGLE RELEVANCE — the candidate serves THIS slot's assigned analytical_story MORE DIRECTLY than the current card, and you can quote the specific word/phrase from the story angle that proves it. A tie or a vague 'better fit' is a KEEP.
2. LISTED — the candidate is one of THIS slot's printed SWAP CANDIDATES (the only size-equivalent, render_real options). A card listed elsewhere or not at all is NOT a valid target.
3. NO DUPLICATE — the target must be NEW to the page: never a card already on the page, already in 1a's TEMPLATE CARD SET (template_card_ids), or already chosen as another slot's target. 1a's chosen cards are sacred.
INTERDEPENDENCY: a card printing link_type master-selector / shared-selection / cross-highlight is COUPLED — swappable ONLY as an all-or-nothing COMBO naming every linked partner's matching swap in `cascade`; if any partner has no valid listed candidate, KEEP the whole set.
Set action='swap' ONLY when all three hold AND confidence >= 0.9 AND `criterion` NAMES the concrete story-angle word the candidate serves better (vague criteria — 'better', 'more relevant', 'nicer', 'good fit' — score below 0.9 and become a KEEP).
