== exact_metadata (the FINISHED METADATA block — you author it exact) ==
Author the COMPLETE METADATA tier for this card, one key per `metadata_shape` field, BYTE-IDENTICAL to the card's static-config default UNLESS the prompt + 1a story justify a morph. Rules:
- BYTE-IDENTICAL DEFAULT — copy each default verbatim from the card's STATIC CONFIG block shown to you (titles, metricTabs, statusColors, statusLegend, units, descriptors, selectionColors, bandThresholds, HPQ tiles/segments/columns/vocab/palette, …). The resting render must not move; only a deliberate morph moves a pixel.
- ★ HARDCODED / STATIC ITEMS ARE LOCKED — never modify them. Any item that is fixed CONFIG, not story-driven — legends (statusLegend), colour tokens / palettes / statusColors / selectionColors, axis labels (metricAxisLabels), units, descriptors, vocab, playback glyphs, shade/legend rows (e.g. the heatmap FOOTER's time-axis + shade legend), badge labels — is HARDCODED. Emit it BYTE-IDENTICAL to the default ALWAYS; it must NEVER appear in `_morphed`. If the payload shipped a value hardcoded, leave it hardcoded — you do NOT touch it.
- ★ THE ONE EXCEPTION TO LOCKED UNITS — A DECLARED SAME-QUANTITY-FAMILY PROXY MUST MORPH ITS DISPLAY METADATA. ★★ THE CANONICAL PROXY RULE (stated ONCE, here — every other mention of substitutes/proxies defers to this): a substitute/proxy is legal ONLY within the slot's physical-quantity family (the bound column's `qty=` must EQUAL the slot's `expected_qty=`). SAME family: bind it + declare it in `data_note` + morph the caption/label/window leaves that DESCRIBE that slot (list each in `_morphed`). DIFFERENT family: OMIT the field — no data_note legitimizes it; the gate blanks it regardless. Same-family examples: the import register is dead on a reversed-CT feeder, so the energy slot carries the EXPORT register delta; a today's-energy slot carries a coarser this-week delta; per-phase kW absent → total kW. The describing leaves are NOT locked for a declared proxy: you MUST morph them to the proxy's TRUE meaning (e.g. `sourceInputCaption` 'SOURCE INPUT'→'MEMBER IMPORT', a window caption 'today'→'this week'), list each in `_morphed`, and keep `data_note` naming the proxy. Rendering a proxy value under the default caption shows a WRONG reading as truth — a declared proxy without the display morph is a defect. If sibling slots share the morphed leaf but keep their own meaning, morph the per-slot caption instead of the shared leaf. PLAN PROXIES FIRST: exact_metadata is emitted BEFORE data_instructions, so decide every proxy binding before you author the metadata — the gate DROPS a declared proxy whose display metadata was not morphed (the leaf then honest-blanks instead of showing a wrong reading). ★ A CROSS-QUANTITY PROXY IS FORBIDDEN, MORPHED OR NOT — a slot of one physical quantity NEVER carries a column of another (power is not temperature, energy is not a count of days, deviation is not flicker); no display morph legitimizes it, and the QUANTITY WALL gate blanks the bind regardless. When no same-quantity source exists the slot is HONEST-BLANK (see BEST-EFFORT + ANSWERABILITY in the data_instructions part) — there is no unit morph that turns a kW number into anything else.
- MORPH ONLY THE STORY-DRIVEN FEW — the ONLY keys you may change are the handful the prompt/story EXPLICITLY drives: lead with the metric the prompt asks about (tab order), re-title, tighten/loosen a threshold the prompt flags, re-roster to the asked entity. Every morph must quote a word from the page story / this card's story angle. WHEN IN DOUBT, DO NOT MORPH — keep the default. A morph on a locked static item is a defect (the byte-identity gate will reject it).
- ★ PERIOD LABELS MUST MATCH THE FILL WINDOW — a period-declaring metadata leaf (`periodLabel`, `range`, a
  "Monthly"/"Weekly"/"Today" caption; NOT a picker's rangeOptions list) must agree with the window/range your
  data_instructions declare (which is the window the consumer fills from). Keeping a default 'Monthly'/'this-month'
  leaf while fetching a 24h window mislabels every number under it. Either declare the range the label promises, or
  MORPH the label leaf to the fetched window's truth (list it in `_morphed`) — an incoherent leaf is deterministically
  morphed/blanked by the gate (policy `gates.window_label_policy`) and recorded.
- SEED ROSTERS ARE FIXTURES, NOT CONFIG — a metadata leaf that LISTS entity identifiers (panel/feeder display names, meter table ids, entity ids inside a roster array) came from the Storybook fixture, NOT from this run's plant. Those identifier strings are STORY-DRIVEN, never locked: when the run's ASSET / PANEL MEMBERS facts name the real members, re-roster the identifier strings to those verbatim member names (declare each path in `_morphed`). If the real member set is NOT in your facts (or has a different size than the fixture roster), do NOT invent names and do NOT silently present the fixture roster as this asset's — keep the default bytes but SAY SO in `data_note` (e.g. "roster identifiers are fixture defaults; real members not resolved"). Presenting another plant's panel names as this asset's members is a seed leak.
- EVERY METADATA FIELD IS REQUIRED AND ALWAYS POPULATED — never omit a key; an omitted key is a render regression. No nulls where the shape needs a value.
- NEW RENDERERS OPT-IN DEFAULT-OFF — any toggle a renderer reads (e.g. showLegend) defaults to its resting value (false), never on, unless the prompt explicitly asks for it.
- ZERO CHROME — no pixel geometry, no fonts, no Card/grid markup, no functions. Metadata is data-shaped scalars/enums/arrays/hex-strings only.
- DUAL-OWNED SLOTS ('AI-default, data-overridable') — when the user message flags a metadata key of THIS card as ★ DUAL-OWNED, author your DEFAULT for it as usual; the worker MAY overwrite it from a live frame. You still emit the default so the resting render is byte-identical.
- `_morphed` IS REQUIRED — THE ONLY CHANNEL AN AUTHORED CHANGE SHIPS THROUGH: emit `"_morphed": [...]` inside exact_metadata listing EVERY key path you changed off-default (`[]` when you morphed nothing). The producer applies ONLY declared morphs — an UNDECLARED morph is REVERTED to the byte-identical default (a silent no-op, recorded as telemetry), and a declared proxy whose caption morph is undeclared is gate-DROPPED. No `_morphed` entry ⇒ your morph does not exist.
