# EMS Query Copilot — `pipeline_v48/copilot`

A **pre-submission prompt-authoring assistant** wired into the v48 host. As the user
types an EMS query, it returns an inline **autofill** (ghost text) + **top-5 suggestions**
(Google-search-style), grounded in the real **v48** EMS metadata and generated by a
dedicated 4B model. It is a **self-contained layer** — it does **not** import the v48
pipeline runtime (`layer1a/layer1b/layer2/run/validate/...`); enforced by `tests/test_no_coupling.py`.

```
{ "autofill": "show transformer 1 current trends this week",
  "suggestions": ["compare current loading between transformer 1 and 2", ...] }
```

## Architecture (pure-AI, retrieve-then-generate)

```
keystroke ─▶ server.py (:8772) ─▶ retrieve.py ─▶ copilot_index.sqlite   (real v48 EMS entities)
                  │                     │
                  └──▶ generate.py ─▶ Qwen3-4B-Instruct-2507-FP8 @ :8201 ─▶ {autofill, suggestions}
```

- **retrieve.py** grounds the model in real entities (assets, salient metrics, relevant
  cards/pages, exemplar questions, real time presets) — no invented assets/metrics.
- **generate.py** is pure AI: the 4B writes the autofill + 5 suggestions from that grounding.

## v48 data sources (read-only; its own `psql` helper, no pipeline imports)

- `cmd_catalog` (rebuilt for v48) — **145 cards**, **75 pages/templates**, 45 derived metrics,
  12 areas, `card_data_recipe` (metric salience), `card_controls` (time presets). Deep semantics
  mined into **440 exemplar questions** (`user_question`/`sem_answers`/`reusable_answers`) + rich keywords.
- **`target_version1.neuract.device_mappings`** — the v48 live asset+metric universe:
  distinct `table_name` = **317 assets**, distinct `field_key` = **234 metrics**.
- `lt_panels_db.lt_mfm` — friendly-name **bridge** for the electrical meters
  (`mfm_NNN` ↔ `cmp_mfm_NNN` → "Transformer 1"); BMS assets get title-cased labels ("Ahu 5").

Index: **1286 entities** (317 asset · 279 metric · 145 card · 440 question · 75 page ·
12 area · 18 time) + curated/embedded/LLM aliases. All DB names live in `config.py`.

## Files (atomic — one concern per file)

| File | Role |
|---|---|
| `config.py` / `db.py` / `llm.py` | DB names + knobs · read-only psql reader · stdlib LLM client (:8201) |
| `aliases.py` | curated synonyms + name cleaning + build-time LLM alias generation |
| `build.py` | builds `copilot_index.sqlite` from the v48 DBs (`--llm` adds colloquial aliases) |
| `retrieve.py` | in-memory grounding (prefix/word/substring/fuzzy/alias + class-salient metrics) |
| `prompts.py` / `generate.py` | rich grounded prompt + pure-AI generation + light guardrails |
| `server.py` | standalone HTTP (:8772): `/`, `/copilot/health`, `POST /copilot/suggest` |
| `frontend/demo.html` | self-contained ghost-text + dropdown demo (served at `/`) |
| `deploy/*.service` | user-systemd units for the model (:8201) and API (:8772) |
| `tests/` | `test_no_coupling.py` (no v48 pipeline imports), `eval.py` (golden prompts) |

## v48 host integration (the live wiring)

- **Frontend:** `pipeline_v48/host/web/src/components/PromptBar.tsx` — Google-style ghost-text +
  suggestions dropdown (Tab/→ accept ghost, ↑/↓ preview, Enter run, Esc restore, bold completion).
  Submit contract (`onRun`) unchanged.
- **Proxy:** `host/web/vite.config.ts` proxies `/copilot` → `http://localhost:8772`
  (env `V48_COPILOT_API`). The widget POSTs relative `/copilot/suggest`.
- **No host/server.py (:8770) change** — the copilot stays a separate service.

## Run

```bash
# 1. model (dedicated user unit; separate from the pipeline's 35B on :8200)
#    32k context + n-gram speculative decoding (output reuses prompt tokens -> faster)
systemctl --user start vllm-copilot          # vllm serve Qwen/Qwen3-4B-Instruct-2507-FP8 --port 8201 \
                                              #   --gpu-memory-utilization 0.17 --max-model-len 32768 \
                                              #   --speculative-config '{"method":"ngram","num_speculative_tokens":5,"prompt_lookup_max":5,"prompt_lookup_min":2}'
# 2. (re)build the v48 index
python3 build.py --llm
# 3. API + demo
python3 server.py                             # http://localhost:8772/  ·  POST /copilot/suggest
# 4. v48 host frontend (proxies /copilot → :8772)
cd ../host/web && npm run dev                 # http://localhost:5188/

# tests
python3 tests/test_no_coupling.py
python3 tests/eval.py
```

## Latency & caching

- Uncached ≈ **0.5–0.7 s/call** (4B @ 32k + n-gram spec-decode; ~98% is decode, prefill ~18 ms).
- **Three caching layers** keep it feeling instant:
  1. **Client cache** (`PromptBar`, per-prefix Map) — revisiting a prefix (backspace/retype) is 0-network.
  2. **Optimistic ghost** — the inline completion stays visible while you keep typing into it
     (no blank-per-keystroke), recomputed from the last known completion.
  3. **Server cache** (`server.py`, per-text) — shared across sessions.
- The widget debounces ~160 ms and cancels stale requests; ghost is derived from `autofill`
  OR the first suggestion that continues the typed text (so it appears consistently).
- Further latency headroom (not yet applied): split a tiny autofill-only call for the ghost,
  or stream the suggestions.
- Pure-AI for now. If the model extrapolates an asset/panel name not in the index, add a
  deterministic entity-validation guard (the deferred step).
